# JavaScript?

## 1. 정의
- 웹 페이지에 생동감을 불어넣기 위해 만들어진 프로그래밍 언어
- 스크립트(script): 자바스크립트로 작성한 프로그램
- 스크립트는 웹 페이지의 HTML 안에 작성할 수 있음
- 웹 페이지를 불러올 때 스크립트가 자동으로 실행됨
- JavaScript는 서버에서도 실행 가능
- JavaScript 엔진 프로그램이 들어있는 모든 디바이스에서 동작

---
- 브라우저에 JavaScript  virtual machine 엔진 내장됨
  - V8: Chrome, Opera
  - SpiderMonkey:Firefox
- 엔진의 기본 동작 원리
  1. 엔진이 스크립트 읽음 (parsing)
  2. 읽은 스크립트를 기계어로 전환함 (compile)
  3. 기계어로 전환된 코드가 실행됨

## 2. 브라우저에서 할 수 있는 일
- Node.js 환경에서 JavaScript는 임의의 파일을 읽거나 쓰고 네트워크 요청을 수행하는 함수를 지원함
- 브라우저 환경에서 JavaScript는 웹 페이지 조작, 클라이언트와 서버 상호작용에 관한 일을 함
  - 페이지에  새로운 HTML 추가하거나 기존 HTML, 스타일 수정
  - 마우스 클릭 등 사용자 행동에 반응
  - 네트워크를 통해 원격 서버에 요청, 파일 다운로드 및 업로드
  - 쿠키를 가져오거나 설정, 사용자에게 질문 및 메세지 보여주기
  - 클라이언트 측에 데이터 저장 (로컬 스토리지)

## 3. 브라우저에서 할 수 없는 일
- 브라우저는 사용자 보안을 위해 JavaScript 기능에 제약을 걸어놓음
- 웹 페이지 내 스크립트는 디스크에 저장된 파일을 읽고 쓰는 접근에 제약 받을 수 있음 
  - input 태그 등 특정 상황에만 파일 접근 허용
- 브라우저 내 탭과 창은 서로 정보를 알 수 없음 
  - JavaScript를 사용해 창에서 다른 창을 열 때는 예외 
  - But, 도메인이나 프로토콜이 다르다면 페이지 접근 불가
  - Same Origin Policy: 동일 출처 청책

## 4. JavaScript만의 강점
1. HTML/CSS와 완전히 통합할 수 있음
2. 간단한 일은 간단하게 처리하게 해줌
3. 모든 주요 브라우저에서 지원하고 기본 언어로  사용됨 (→ 브라우저 인터페이스를 만들 때 가장 널리 사용)

## 5. JavaScript 너머의 언어들
- 브라우저가 실행되기 전에 JavaScript로 transfile(변환)할 수 있는 언어들
  - CoffeeScript, Flow, Dart
  - TypeScript: 자료의 명시화 (strict data typing)
